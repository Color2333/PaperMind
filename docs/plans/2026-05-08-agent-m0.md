## 背景

用户实测发现 agent 无法完成「把 5 月份入库的论文全部粗读一遍」这种再普通不过的需求：

- `search_papers` 只搜文本字段（title/abstract），传 `keyword=2025-05` 永远 0 篇
- agent 自己说「无法直接按入库日期筛选论文」 — **工具集缺口**
- 单篇 `skim_paper` 工具带 confirm token，**批量场景下来不及全确认完就过期**
- 输入 `b7e07388` 短 ID 直接报「无效的 paper_id 格式」 — DX 灾难

→ 不是 LLM 笨，是工具集结构性缺失。

## M0 目标

让用户能说：
> 「把 5 月份入库的所有论文都粗读一遍」

agent 一次确认 → 后台跑完 → 回报。

## 已敲定的决策（不要再改）

| 决策点 | 方案 | 理由 |
|---|---|---|
| 筛选实现 | 直接给 `PaperRepository.list_paginated` 加 `start_date` / `end_date` / `date_field` 三个参数 | 现有方法已支持 90% 维度（topic_id/status/tag_ids/category/search/sort），只缺日期范围 |
| 批量调度 | API 进程内 daemon thread 做 consumer，5 秒扫一次 `batch_jobs` 表 | 不影响 worker 容器；用户期望 5 秒内开始处理；幂等靠启动时 `running → failed` |
| 错误处理 | 跳过失败项继续跑完，最后返回失败清单 | 100 篇里 1 篇网络抖动不该让 99 篇陪葬 |
| 短 ID 阈值 | 最少 8 位前缀 | UUID 前 8 位碰库概率极低；`_parse_uuid` / `_require_paper` 改造 |
| 不复用 PipelineRun | 新建独立 `BatchJob` 表 | 语义不同（单批 vs 单篇）|
| 不动前端 | 仅后端 + agent prompt | M0 范围控制；前端可以下个 M 再做进度可视化 |

### 2. `BatchJob` 模型 + alembic migration

**文件**：`packages/storage/models.py`（追加）

```python
class BatchJob(Base):
    __tablename__ = "batch_jobs"
    id: Mapped[str] = mapped_column(String(36), primary_key=True, default=lambda: str(uuid4()))
    kind: Mapped[str] = mapped_column(String(32), nullable=False, index=True)  # skim/deep_read/embed
    status: Mapped[str] = mapped_column(String(16), nullable=False, default="pending", index=True)
    # pending → running → completed | failed
    total: Mapped[int] = mapped_column(Integer, nullable=False, default=0)
    done: Mapped[int] = mapped_column(Integer, nullable=False, default=0)
    failed: Mapped[int] = mapped_column(Integer, nullable=False, default=0)
    paper_ids: Mapped[list] = mapped_column(JSON, nullable=False, default=list)
    error_log: Mapped[dict] = mapped_column(JSON, nullable=False, default=dict)
    # {paper_id: error_message}
    created_by: Mapped[str | None] = mapped_column(String(64), nullable=True)  # agent / api
    created_at: Mapped[datetime] = mapped_column(DateTime, default=_utcnow, nullable=False)
    started_at: Mapped[datetime | None] = mapped_column(DateTime, nullable=True)
    finished_at: Mapped[datetime | None] = mapped_column(DateTime, nullable=True)
```

**Repository**：在 `packages/storage/repositories.py` 新增 `BatchJobRepository`：
- `create(kind, paper_ids, created_by) -> BatchJob`
- `get(job_id) -> BatchJob | None`
- `claim_next() -> BatchJob | None`（原子拿一条 pending 改成 running）
- `mark_progress(job_id, done, failed, error_log_patch)`
- `mark_finished(job_id, status)`（completed | failed）

### 3. 批量执行器（API daemon thread）

**新文件**：`packages/agent_core/batch_consumer.py`

```python
import threading, time, logging
from packages.storage.db import session_scope
from packages.storage.repositories import BatchJobRepository, PaperRepository
from packages.ai.pipelines import PaperPipelines
from packages.integrations.llm_client import LLMClient

logger = logging.getLogger(__name__)
_thread = None
_stop = threading.Event()
_POLL_INTERVAL = 5.0

def _run_one_job(job):
    pipelines = PaperPipelines()
    llm = LLMClient()
    for pid_str in job.paper_ids:
        if _stop.is_set():
            break
        try:
            from uuid import UUID
            pid = UUID(pid_str)
            if job.kind == "skim":
                pipelines.skim(pid)
            elif job.kind == "deep_read":
                pipelines.deep_dive(pid)
            elif job.kind == "embed":
                # 复用 _embed_paper 的逻辑
                with session_scope() as s:
                    paper = PaperRepository(s).get_by_id(pid)
                    text = (paper.title or "") + "\n" + (paper.abstract or "")
                    vec = llm.embed_text(text)
                    # TODO: 调用 PaperRepository.update_embedding(pid, vec)
            with session_scope() as s:
                BatchJobRepository(s).mark_progress(job.id, done_delta=1)
                s.commit()
        except Exception as exc:
            logger.warning("batch job %s paper %s failed: %s", job.id, pid_str, exc)
            with session_scope() as s:
                BatchJobRepository(s).mark_progress(job.id, failed_delta=1, error_patch={pid_str: str(exc)[:300]})
                s.commit()

def _consumer_loop():
    while not _stop.is_set():
        try:
            with session_scope() as s:
                job = BatchJobRepository(s).claim_next()
                s.commit()
            if job is None:
                _stop.wait(_POLL_INTERVAL)
                continue
            _run_one_job(job)
            with session_scope() as s:
                BatchJobRepository(s).mark_finished(job.id, "completed")
                s.commit()
        except Exception:
            logger.exception("batch consumer loop error")
            _stop.wait(_POLL_INTERVAL)

def start():
    global _thread
    if _thread and _thread.is_alive():
        return
    # 启动时幂等：把 running 状态的恢复成 failed
    with session_scope() as s:
        BatchJobRepository(s).recover_stale_running()
        s.commit()
    _thread = threading.Thread(target=_consumer_loop, daemon=True, name="batch-consumer")
    _thread.start()
    logger.info("batch consumer thread started")

def stop():
    _stop.set()
```

**接入**：在 `apps/api/main.py` 的 lifespan / startup 里调 `batch_consumer.start()`，shutdown 里调 `stop()`。

### 5. 短 paper_id 模糊匹配

**文件**：`packages/ai/agent_tools.py:30-50`

```python
def _resolve_paper_id(val: str) -> tuple[UUID | None, str | None]:
    """返回 (uuid, error_msg)。支持完整 UUID 或 ≥8 位前缀模糊匹配。"""
    val = (val or "").strip()
    # 试完整 UUID
    try:
        return UUID(val), None
    except ValueError:
        pass
    # 前缀模糊匹配（要求 ≥8 位）
    if len(val) < 8 or not all(c in "0123456789abcdef-" for c in val.lower()):
        return None, "无效的 paper_id 格式（需完整 UUID 或 ≥8 位 hex 前缀）"
    with session_scope() as s:
        # 用 LIKE 'val%' 找
        from sqlalchemy import select
        from packages.storage.models import Paper
        rows = list(s.execute(
            select(Paper.id).where(Paper.id.ilike(f"{val}%")).limit(2)
        ).scalars())
    if not rows:
        return None, f"未找到匹配 '{val}' 的论文"
    if len(rows) > 1:
        return None, f"前缀 '{val}' 匹配多篇论文，请使用完整 UUID"
    return UUID(rows[0]), None

def _require_paper(paper_id: str):
    pid, err = _resolve_paper_id(paper_id)
    if err:
        return None, ToolResult(success=False, summary=err)
    with session_scope() as session:
        try:
            paper = PaperRepository(session).get_by_id(pid)
            return paper, None
        except ValueError:
            return None, ToolResult(success=False, summary=f"论文 {paper_id[:8]}... 不存在")
```
