## 背景

GitHub issue #19 反馈：开源项目 README 转化漏斗里 95% 的访客不会 `git clone`，只是想 30 秒验证一下「PaperMind 读论文到底是什么体验」。

第三方平台（如 socialistic.ai）打包的「demo」只是把 agent-loop skill 拿出去跑，**不是 PaperMind 的真实体验**——核心价值（FastAPI 后端 + pgvector 检索 + 标签系统 + 论文导入 + agent 流程）必须自建 demo 站才能呈现。

→ 自建一个**公网可访问、零安装、只读体验**的 demo 站，让访客不 clone 不配环境就能体验「导入论文 → 自动打标签 → agent 总结/笔记」核心流程，并在 README 顶部贴入口。

## 已敲定的决策（不要再改）

| 决策点 | 方案 | 理由 |
|---|---|---|
| 部署平台 | **Vercel（前端） + Railway（后端 + Postgres）** | Vercel CDN 全球前端加速；Railway 支持 Dockerfile + 自带 Postgres + pgvector |
| 数据策略 | **预灌 30 篇 arXiv + 只读体验** | 零滥用风险；CS.AI / CS.CL / CS.CV 各 10 篇近期热门 |
| LLM Provider | **智谱 GLM-4-Flash**（免费，有 RPM 限制） | 项目方零成本；现有 `llm_client.py` 已支持 `LLM_PROVIDER=zhipu`，零代码改动 |
| Embedding | **智谱 embedding-3**（与 chat 同一 key） | 少配一个 key，少一处费用；不用阿里百炼 DashScope |
| 限流存储 | MVP 用进程内字典 + asyncio Lock | YAGNI，二期再升 Redis |
| Worker / 定时任务 | **不跑** scheduler / cs_feed_orchestrator | demo 站纯被动响应，无后台拉取 |
| 域名 | 大白已有，待提供具体子域名 | 例 `demo.papermind.xxx` + `api.demo.papermind.xxx` |
| 上线时间 | 本周内 MVP | 优先打通最小闭环，后续迭代 |

## 不做的事（YAGNI 严控）

- ❌ 用户系统 / 登录

- ❌ Cloudflare Turnstile（先观察刷量再决定）

#### 1.2 新增 DemoMode 中间件（4-5h）

**新文件 `apps/api/middleware/__init__.py`**：空
**新文件 `apps/api/middleware/demo_mode.py`**：

```python
"""
PaperMind Demo 模式中间件
@author Color2333

功能：
1. DEMO_MODE=true 时拦截所有写接口（POST/PUT/DELETE/PATCH），返回 403
2. 按 IP 限流（默认 30 req/h）
3. 全局 RPM 闸门（默认 50/min，保护智谱 key）
4. 失败时返回友好降级响应
"""

from __future__ import annotations
import os
import time
import asyncio
from collections import defaultdict, deque
from fastapi import Request
from fastapi.responses import JSONResponse
from starlette.middleware.base import BaseHTTPMiddleware


# 写接口白名单（demo 模式下仍允许）
WRITE_WHITELIST_PREFIX = (
    "/agent/chat",      # agent 对话必须能 POST
    "/agent/skim",      # 查看预灌论文的总结
    "/papers/search",   # 检索接口
)


class DemoModeMiddleware(BaseHTTPMiddleware):
    def __init__(self, app, ip_limit_per_hour: int = 30, global_rpm: int = 50):
        super().__init__(app)
        self.enabled = os.getenv("DEMO_MODE", "false").lower() == "true"
        self.ip_limit = ip_limit_per_hour
        self.global_rpm = global_rpm
        self._ip_buckets: dict[str, deque[float]] = defaultdict(deque)
        self._global_bucket: deque[float] = deque()
        self._lock = asyncio.Lock()

    async def dispatch(self, request: Request, call_next):
        if not self.enabled:
            return await call_next(request)

        method = request.method
        path = request.url.path

        # 1. 写接口白名单外，403
        if method in {"POST", "PUT", "DELETE", "PATCH"}:
            if not any(path.startswith(p) for p in WRITE_WHITELIST_PREFIX):
                return JSONResponse(
                    status_code=403,
                    content={
                        "detail": "Demo 站只读体验，完整功能请克隆部署: https://github.com/Color2333/PaperMind"
                    },
                )

        # 2. IP + 全局限流
        client_ip = request.client.host if request.client else "unknown"
        now = time.time()
        async with self._lock:
            # IP 桶（1 小时窗口）
            bucket = self._ip_buckets[client_ip]
            while bucket and bucket[0] < now - 3600:
                bucket.popleft()
            if len(bucket) >= self.ip_limit:
                return JSONResponse(
                    status_code=429,
                    content={"detail": f"请求过于频繁，每 IP 每小时上限 {self.ip_limit} 次"},
                )
            bucket.append(now)

            # 全局桶（1 分钟窗口）
            while self._global_bucket and self._global_bucket[0] < now - 60:
                self._global_bucket.popleft()
            if len(self._global_bucket) >= self.global_rpm:
                return JSONResponse(
                    status_code=503,
                    content={"detail": "demo 站当前繁忙，请 1 分钟后重试"},
                )
            self._global_bucket.append(now)

        return await call_next(request)
```

### Day 2：数据预灌脚本

#### 2.1 写 `scripts/seed_demo.py`（5-6h）

**目标**：从 arXiv 拉 30 篇精选论文（CS.AI / CS.CL / CS.CV 各 10），跑完整 pipeline 入库。

**结构骨架**：
```python
"""
Demo 站数据预灌脚本
@author Color2333

用法:
  DEMO_MODE=false python scripts/seed_demo.py --count 30 --dry-run  # 先看会拉哪些
  DEMO_MODE=false python scripts/seed_demo.py --count 30            # 实际入库
"""
import argparse
from packages.integrations.arxiv_client import ArxivClient
from packages.ai.pipelines import PaperPipelines

CATEGORIES = ["cs.AI", "cs.CL", "cs.CV"]
PER_CATEGORY = 10

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--dry-run", action="store_true")
    args = parser.parse_args()

    client = ArxivClient()
    pipelines = PaperPipelines()

    for cat in CATEGORIES:
        # 1. 拉近 30 天该 category 引用 / 关注度高的论文
        papers = client.search(category=cat, max_results=PER_CATEGORY * 2, sort="relevance")
        for arxiv_paper in papers[:PER_CATEGORY]:
            if args.dry_run:
                print(f"[dry] {cat}: {arxiv_paper.title}")
                continue
            paper_id = pipelines.ingest_from_arxiv(arxiv_paper)  # 入库
            pipelines.embed_paper(paper_id)                      # 向量化
            pipelines.skim(paper_id)                             # 自动总结
            pipelines.auto_tag(paper_id)                         # 自动打标签
            print(f"[done] {paper_id} {arxiv_paper.title}")

if __name__ == "__main__":
    main()
```

**关键点**：
- 看 `packages/integrations/arxiv_client.py` 现有 search 方法的真实签名，按需调整
- `PaperPipelines.ingest_from_arxiv` 等方法名以仓库实际为准（看 `packages/ai/pipelines.py`）
- 失败重试：单篇失败不影响后续；记录到 `data/seed_failed.txt`
- **dry-run 必须先跑一遍**，避免一次性把 key 配额耗光

| 风险 | 影响 | 应对 |

| 智谱 key 被刷爆 | demo 不可用 | 全局 RPM 闸门 + IP 限流 + 失败时降级返回 DB 缓存的旧总结 |
